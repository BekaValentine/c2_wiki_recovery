Mess As Excuse For Neglecting Code Browsers

Many people regard CodeBrowserAsExcuseForMess, but they are an important tool for refactoring huge masses of already existing code to fit them in a new structure. Neglecting to do so ultimately leads to rewriting the mass of code and so CodeDuplication.

Given the premise of the previous paragraph, shouldn't this page be renamed to MessAsExcuseForCodeBrowsers?
