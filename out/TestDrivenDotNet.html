Test Driven Dot Net

BrianButton recommended using the Test'Driven.NET library (TestDrivenDotNet) to make it easier to run tests within the Microsoft VisualStudioTeamSystem environment.

Tool Home: http://www.testdriven.net/

Brian Button's "Real Test Driven Development with Visual Studio Team System" article: http://www.agileprogrammer.com/oneagilecoder/archive/2005/11/28/9859.aspx

Provides VisualStudio support for NCover, NCoverExplorer, Reflector, TypeMock, dotTrace, NUnit (NunitFramework), MbUnit, ZaneBug, MSBee & TeamSystem.
