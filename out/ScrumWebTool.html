Scrum Web Tool

The first brainstorm

Project Description SCRUM is a software project management methodology. Its major benefits are the increase of team productivity, the increase of software quality and the meet of all the goals. Business Requirement and Non Functional statements are every day observed. The Web Based tool for managing scrum help project manager, team members and project owners to focus on their commitments.

Features:

  • Web User Interface;
  • Multi user account;
  • Multi project scalability;

For each project the features are:

  • Product Backlog;
  • Sprint Backlog;
  • Sprint managements;
  • Sprint goal definition;
  • Team Planner;
  • Issue management;
  • Project/Sprint budget estimation;
  • Project/Sprint budget estimation evaluation considering drag factors and scenarios
  • Reporting (Work Remaining Graph, and others)
  • User account management (Team members, Scrum master, Project Owner)
  • Daily team backlog management
  • Integration with Microsoft Excel, with Microsoft Project and with Open Office
  • Scheduling of Virtual Scrum
  • Mobile Daily Task management for the team member
  • Sashimi management

Non functional requirement:

  • must be able of scaling across platform;
  • must be able on running on open source platform;
  • must run on the Web;
  • must support the whole range of Database
  • must expose inquiry model as Web Service

Wishlist:

  • allow addition of user-specific columns (to be used in site-specific metrics). Things like "Workorder Number" and "Feature type" (product or infrastructure) are things I have added to my own spreadsheet, for later correlation with other systems at my workplace. Being my first sprint, I've already added a dozen columns to help us measure the benefits of using Scrum vs. our other ways of working. Hopefully, they'll disappear over time... -- DebHartmann July 19, 2003
